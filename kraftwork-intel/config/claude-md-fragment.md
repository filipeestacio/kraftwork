## Memory (kraftwork-intel)

- At the start of any task involving an unfamiliar codebase area, call `memory-recall` with relevant terms before reading files.
- After resolving a non-obvious bug, understanding an architectural decision, or discovering an undocumented pattern, call `memory-memorize` to store the learning.
- Categories to use: `architecture`, `patterns`, `debugging`, `decisions`, `conventions`.
- Learnings should be specific and reusable — not "fixed a bug" but "the event bus uses correlation IDs for request tracing; always propagate X-Correlation-ID through service calls".
