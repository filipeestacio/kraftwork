## Messaging (Slack)

- Use `messaging-find` to locate the right channel before sending anything.
- Use `messaging-describe` to read recent channel context before posting updates — avoid duplicating messages already sent.
- Send notifications proactively when: a merge request is created, a pipeline fails, or a deployment completes.
- Match the channel to its declared purpose — check the `channels` list in workspace.json before routing a message.
